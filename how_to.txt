===========================================================
  EVE PI TEMPLATE GENERATOR — HOW TO USE
===========================================================

-----------------------------------------------------------
GETTING STARTED
-----------------------------------------------------------

Double-click "Eve PI.exe" to launch the app.
  OR
Run from source:  python PI.py

Keep the "data" folder next to the .exe — the planet icons,
template library and planet radius table are read from
there, not from inside the executable.

The main window has six configuration steps, a Bill of
Materials panel below them, and three buttons at the bottom:
GENERATE TEMPLATE, PROXIMITY SCOUT and TEMPLATE LIBRARY.

-----------------------------------------------------------
STEP 1 — GENERATING A TEMPLATE
-----------------------------------------------------------

Work down the numbered panels:

  1. PRODUCT
     Pick what you want the planet to produce. The dropdown
     lists every PI commodity with its tier, e.g.
     "Robotics  [P3]", sorted P1 → P4.
     Choosing a product decides which chains are offered
     next, so always set this first.

  2. CHAIN
     How the product gets made. Only chains that can produce
     the selected product are listed:

       P0 → P1 (Extraction)
         Extractor + basic factories. The classic mining
         planet.

       P0 → P2 (Extraction)
         Self-sufficient planet: extracts its own raw
         resources, refines them to P1, then to P2. If the
         planet only holds one of the two raw resources it
         still works — the other P1 is imported.

       P1 → P2 (Factory)
         Import P1, produce P2.

       P1 → P3 (Factory)
         Two-stage planet: one bank of factories turns P1
         into the P2 intermediates, a second bank turns
         those into P3. You only haul P1.

       P2 → P3 (Factory)
         Import finished P2, produce P3. Far more output
         than P1 → P3, but more hauling.

       P1 → P4 (Factory)
         Deepest chain: P2 and P3 factories on the planet
         feed a single High-Tech facility. Least hauling,
         lowest output.

       P2 → P4 (Factory)
         Import P2, build P3 on-planet, assemble P4.

       P3 → P4 (Factory)
         Import finished P3, assemble P4. Highest P4 output,
         most hauling.

  3. PLANET TYPE
     Only valid planet types are offered:
       - P4 chains are limited to Barren and Temperate,
         the only types that allow a High-Tech facility.
       - Extraction chains list the planet types that
         actually hold the raw resources, with the most
         complete ones first.

  4. PLANET RADIUS (km)
     Type the radius of your actual planet. Find it in-game:
     select the planet, then Information → Attributes →
     Radius. Defaults to 5000. This only scales the layout;
     it does not change the structure count.

     On a P0 → P1 planet a checkbox also appears here:
     "Include Storage Facility (P0 buffer)". It puts a
     Storage Facility between the extractor and the
     factories, so raw material that arrives faster than the
     factories consume it has somewhere to sit instead of
     being lost. Worth it if you run short extraction
     programs that spike well above the average yield.

  5. COMMAND CENTER LEVEL
     Your Command Center Upgrades skill (0–5), as clickable
     badges. Higher levels mean more CPU and power, so the
     generator has more to work with.

  6. LAYOUT
     There is no single correct PI layout, so the decisions
     that actually shape a colony are yours:

     Collect every — how long the colony must run without
       you. The generator adds launch pads, and if need be
       drops factories, until the pads can hold that many
       hours of materials. Ask for 48h and you get a smaller,
       slower colony you can ignore for two days; ask for 6h
       and you get maximum output you must service often.

       This bites hardest on factory planets, where the
       imports alone fill a pad quickly. A P3 → P4 planet at
       level 5 goes from 16 facilities on 2 pads at 6h to 8
       facilities on 4 pads at 24h. On extraction planets it
       usually changes nothing: P1 is compact enough that a
       single pad already holds several days of output — a
       balanced level 5 extraction planet runs about 150
       hours untended — so don't be surprised when the layout
       stays put.

     Extractor yield — units per head per hour, shown only
       for extraction chains. This is the number the whole
       chain is balanced against: one basic factory eats
       6000 units an hour, so a 10-head extractor at 2000
       feeds three of them. Raise it if your deposits are
       rich, lower it if the extractors run dry.

     Set counts myself — tick this to place exact numbers of
       extractors, heads, factories and pads. 0 means "you
       decide". Manual values are placed as asked, even bad
       ones: the panel then tells you what is wrong instead
       of silently refusing. Available for the extraction and
       single-stage factory chains; the multi-stage layouts
       (P1→P3, P1→P4, P2→P4) are fixed by geometry.

     The strip at the bottom of the panel measures whatever
     is currently configured, updating as you type:

       CPU and power bars against your command center,
         green until 90%, amber to 100%, red beyond it.
       "runs Nh untended" — green if it meets the interval
         you asked for, amber if it falls short.
       "extract N/h · factories use N/h" on extraction
         chains — red when the factories outrun the ground.
       Any number of ⚠ warning lines in red.

     Your interval and yield choices are remembered between
     sessions; the manual counts are not.

  Then click GENERATE TEMPLATE.

-----------------------------------------------------------
STEP 2 — READING THE OUTPUT
-----------------------------------------------------------

A popup window appears with:

  - The JSON template text
  - A visual planet map of the layout
  - A "Copy JSON" button

To import into EVE Online:
  1. Click "Copy JSON"
  2. In EVE, open the planet's Planetary Interaction view
  3. Use the in-game template import and paste

The map has a legend in the corner covering Launch Pad,
Storage Facility, Basic Industry, Advanced Industry,
High-Tech Industry and Extractor (ECU).

Drag to pan the map, scroll to zoom, and drag the popup
edges to resize. Hover any structure for a tooltip listing
what it consumes and produces per cycle. On pure factory
planets the Launch Pad tooltip also estimates how long it
takes to fill up, so you know how often to collect.

-----------------------------------------------------------
BILL OF MATERIALS
-----------------------------------------------------------

The BOM panel below the configuration steps shows, for one
production cycle:

  - The facility used and its output per cycle
  - INPUTS — what the final factory consumes
  - Either "EXTRACTED ON-PLANET" (green) for what the planet
    mines itself, or "SEND TO LAUNCH PAD" (orange) for what
    you must haul in

On a P0 → P2 planet you may see both: the raw resource the
planet holds is listed as extracted, and the P1 it cannot
make locally is listed as an import.

Rows are colour-coded by tier — P0 grey, P1 pale blue,
P2 green, P3 sand, P4 in the theme's accent colour.

-----------------------------------------------------------
PROXIMITY SCOUT
-----------------------------------------------------------

Finds the planets around a system so you can pick where to
set up.

  1. Click "PROXIMITY SCOUT" at the bottom of the main
     window.
  2. Type a system name — autocomplete suggests matches
     after two characters.
  3. Set JUMPS (0–10) for how far out to scan.
  4. Click SCAN, or press Enter.

Results are grouped by system, closest first. Each system
header shows its security status, the planet types it
contains, its jump distance and its planet count. Click a
header to expand or collapse its planets.

Each planet card shows the planet's CCP icon, its name, its
type, and its radius in km — bigger radius means extractor
heads spread further apart and better yield.

The SHOW row filters by planet type. Click a type to toggle
it, or ALL to bring everything back. Filtering redraws
instantly from the scan you already ran and automatically
expands the systems that still have matches, so you can hunt
for, say, every Lava planet within 5 jumps in one glance.
Your selection is remembered for next time.

The first scan of an area takes a while — a few jumps means
hundreds of ESI requests. Results are cached locally, so
re-running the same system and jump count is instant. The
cache is kept for 7 days.

-----------------------------------------------------------
TEMPLATE LIBRARY
-----------------------------------------------------------

"TEMPLATE LIBRARY" opens 83 bundled DalShooth templates,
grouped into Miner (0.0), Miner (Low-Sec) and Factory.
Double-click one to open it in the same preview window used
for generated templates, then copy it as usual.

-----------------------------------------------------------
SETTINGS
-----------------------------------------------------------

Click the gear icon in the title bar.

  Opacity — window transparency
  Theme   — 23 EVE faction colour schemes

Click APPLY to confirm. Window positions and sizes are saved
automatically as you move and resize them.

The "?" button opens the About panel.

-----------------------------------------------------------
SYSTEM TRAY
-----------------------------------------------------------

Click the minimize button (—) to hide the main window to the
system tray.

  Single-click tray icon  — bring the window back
  Right-click tray icon   — menu with Show / Exit
  Double-click title bar  — collapse/expand the window

-----------------------------------------------------------
TIPS
-----------------------------------------------------------

- Chains within a tier trade hauling against output. For P4:
  P1 → P4 hauls the least and yields one High-Tech facility;
  P3 → P4 hauls finished P3 and yields many. P2 → P4 sits in
  between. The same applies to P1 → P3 versus P2 → P3.

- P0 → P2 is the low-maintenance option: a planet that feeds
  itself needs no imports at all when it holds both raw
  resources.

- Run the Proximity Scout before committing to a system, and
  use the type filter to find the planet types your chain
  needs.

- Radius matters. Check it in-game and type the real value
  before generating.

- Extraction planets are power-limited, not CPU-limited. A
  10-head extractor burns 8100 of the 19000 power a level 5
  command center has, which is why a balanced extraction
  colony leaves CPU unused. Spending that spare power on a
  second extractor leaves almost nothing for factories.

- If the layout panel says the extractors are short, either
  build fewer factories or accept that they idle part of the
  time. Factories that outrun their supply do not produce
  more, they just cost CPU and power.

-----------------------------------------------------------
TROUBLESHOOTING
-----------------------------------------------------------

"Could not generate template"
  — The Command Center level is too low for that layout.
    Raise it, or choose a chain with fewer structures.

"Extractors N/h short of what the factories eat"
  — More factories than the ground can feed. Either lower the
    factory count, add an extractor if power allows, or raise
    the Extractor yield if your deposits really are that rich.
    Starved factories cost CPU and power without producing.

"Storage only lasts Nh, not the Nh asked for"
  — The colony cannot buffer the interval you picked. Add a
    launch pad, drop a factory, or accept a shorter interval.
    In automatic mode the generator has already tried; the
    warning means even its best attempt fell short.

"CPU / Power over budget"
  — Only reachable with manual counts. The template is still
    produced so you can see the shape of it, but EVE will
    refuse to build it as-is.

The layout panel is greyed out
  — Manual counts are only available for the extraction and
    single-stage factory chains. P1 → P3, P1 → P4 and
    P2 → P4 build a fixed geometry that cannot be resized.

Radius shows "—" in the Proximity Scout
  — The planet radius table is missing. The app downloads it
    once from the Fuzzwork SDE dump (~85 MB, several
    minutes) and caches it in data/planet_radii.json. Make
    sure that file is present next to the executable.

Planet cards show plain coloured circles instead of icons
  — data/planet_icons is missing next to the executable.

"No systems found" / scan errors
  — Check your internet connection; the scout reads live
    data from ESI.

App crashes on launch (from source)
  — Install the dependencies: pip install -r requirements.txt
  — pystray is optional, but required for tray support.
